Feedback (Contacting Me)

There are several ways to tell me something, and all of them are valid. Feel free to just use the easiest method for you. Depending on what you want to say, however, one method might be more effective than the others.

Recommendations:

No matter what you want to do...
You can do it by emailing me at tyler.wayne.dev@gmail.com. I'll respond as fast as I can, and I'll make sure that I hear whatever it is that you have to say.
Submitting a bug report...
Is best done through the issue tracking system of the appropriate GitHub project (repository). These allow me to categorize issues effectively, even in large numbers. They are publicly visible, which means that others besides me have the ability to help out.
Suggesting a feature...
Is also a good reason to use the issue trackers mentioned above. The system can track features, bugs, version goals — just about everything that can be categorized (and then some).
Praising/complimenting my work...
If you've got a glowing review, send that to the appropriate Chrome Web Store page for the program. If there is no Web Store page, just email it to me, and (with your permission) I may decide to post it here on the website. Oh, and all of this applies to negative feedback, too. I do want to know how to improve myself and my work.
Offering a job...
Hey, I can hope, right? I'm currently unemployed, but thankfully, I'm not in desperate need of a job. If you're looking, though, I might be interested. Please, email me so that we can talk about it.
